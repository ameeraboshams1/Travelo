What is Travelo?

A focused journey: search → compare → book → manage

Idea

Travelo removes noise from booking. We prioritise clarity (totals without surprises), speed (fewest steps), and control (filters that actually matter). The frontend is plain, readable HTML/CSS/JS; the backend is a small PHP layer with predictable responses.

We care about honest states: draft, paid, ticketed, refunded — each one visible and explainable to the user.

Product Pillars

The principles guiding every screen

Clarity

Totals, policies, and refund reasons are never hidden or delayed.

Speed

Three steps to checkout. Predictable inputs. No reload jumps.

Control

Filters that matter: duration, stops, time windows, cabin class.

Trust

Readable states; idempotent booking flow on the PHP side.

How it Works

Simple steps with explicit states

1. Search
2. Compare
3. Checkout
4. Confirm
User enters from/to/date/class → JS validates and builds a request for PHP search.
Results render as tidy cards with honest totals and stable filters.
One-page checkout (traveler, payment, policy) → safe POST to PHP (idempotent).
Confirmation with booking reference and a clear state.
